# File Operations MCP Server

**AI Employee Vault — Gold Tier**

MCP server that gives Claude Code five file-system tools for managing the vault.

---

## Tools

### 1. `search_files`
Search by filename pattern (glob) and optionally by content.

| Parameter | Type | Required | Description |
|---|---|---|---|
| `pattern` | string | Yes | Glob pattern, e.g. `**/*.md`, `Needs_Action/*` |
| `directory` | string | No | Root dir for search (default: vault root) |
| `content` | string | No | String to search inside file contents |
| `case_sensitive` | boolean | No | Default: `false` |
| `limit` | number | No | Max results (default: 50) |

**Returns:** Array of matching files with path, size, modified date, and matching line previews.

---

### 2. `read_file_content`
Read any vault file. Format-aware: JSON is pretty-printed, CSV metadata included.

| Parameter | Type | Required | Description |
|---|---|---|---|
| `file_path` | string | Yes | Relative to vault root or absolute |
| `max_lines` | number | No | Truncate output to N lines |

**Returns:** Full file content plus metadata (size, line count, word count, modified date).

---

### 3. `create_summary`
Aggregate statistics and auto-generated insights across multiple files or a whole directory.

| Parameter | Type | Required | Description |
|---|---|---|---|
| `file_paths` | string[] | No* | Explicit list of file paths |
| `directory` | string | No* | Summarize all files in this directory |
| `pattern` | string | No | Glob filter when using `directory` (default: `**/*`) |

*One of `file_paths` or `directory` is required.

**Returns:** File count, total size/lines/words, breakdown by extension, per-file stats, insights.

---

### 4. `organize_files`
Move, copy, rename files or create directory structure.

| Parameter | Type | Required | Description |
|---|---|---|---|
| `operation` | string | Yes | `move`, `copy`, `rename`, or `mkdir` |
| `source` | string | No* | Source path (not needed for mkdir) |
| `destination` | string | Yes | Destination path or new directory |

*Required for move, copy, rename.

**Returns:** Operation result with source, destination, and timestamp.

**Notes:**
- Cross-drive moves on Windows automatically fall back to copy + delete.
- Destination parent directories are created automatically.

---

### 5. `analyze_logs`
Parse log files to extract metrics, identify errors and patterns, and assess health.

| Parameter | Type | Required | Description |
|---|---|---|---|
| `log_path` | string | Yes | Path to a log file or directory |
| `pattern` | string | No | Glob for directory scan (default: `**/*.{md,log,txt}`) |
| `filter` | string | No | Only include lines containing this string |

**Returns:**
- Line counts by log level (ERROR, WARN, INFO, DEBUG)
- Top 10 error messages by frequency
- Top 5 recurring patterns
- Time range (earliest → latest timestamp found)
- Health assessment: `HEALTHY` / `DEGRADED` / `CRITICAL`

## Security Notes

- All relative paths are resolved against the vault root.
- Files larger than **10 MB** are skipped for content operations.
- `node_modules/`, `.git/`, and `.obsidian/` directories are excluded from all glob searches.
- All write/move/copy operations are logged to stderr for auditability.

---

## Error Handling

Every tool returns a JSON response with `"status": "success"` or `"status": "error"`.
Errors include a descriptive `"error"` message — no silent failures.

Internal errors are caught at the top-level handler and returned as structured error responses
rather than crashing the server.

# Email MCP Server

MCP (Model Context Protocol) server that gives Claude Code the ability to send emails via the Gmail API.

## Prerequisites

1. **Node.js** v18 or later
2. **Gmail API credentials** — `credentials.json` in the vault root
3. **OAuth token** — `token.json` in the vault root (generated by `gmail_watcher.py` on first run)

## Setup

### 1. Install Dependencies

```bash
cd mcp_servers/email_mcp
npm install
```

### 2. Ensure OAuth Token Exists

The MCP server shares the same OAuth credentials as `gmail_watcher.py`. If you haven't run the watcher yet:

```bash
cd ../..
python gmail_watcher.py
```

This will open a browser for OAuth consent and create `token.json`. You can stop the watcher after authentication completes.

### 3. Configure Claude Code

The MCP config is at `.claude/mcp_config.json` in the vault root. Claude Code will automatically detect and load this server.

To verify the config is loaded, restart Claude Code in the vault directory.

## Available Tools

### `send_email`

Send an email through the authenticated Gmail account.

**Inputs:**
| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `to` | string | Yes | Recipient email address |
| `subject` | string | Yes | Email subject line |
| `body` | string | Yes | Email body (plain text) |
| `cc` | string | No | CC recipients, comma-separated |
| `bcc` | string | No | BCC recipients, comma-separated |

**Returns:** JSON with `status`, `message_id`, `thread_id`, and `timestamp`.

**Example response:**
```json
{
  "status": "success",
  "message_id": "18d1a2b3c4e5f6g7",
  "thread_id": "18d1a2b3c4e5f6g7",
  "to": "recipient@example.com",
  "subject": "Hello from AI Employee",
  "timestamp": "2026-02-18T09:30:00.000Z"
}
```

## HITL Approval Requirement

Per `Company_Handbook.md` and `HITL_APPROVAL_SKILL.md`, **sending any email requires human approval**. The workflow is:

1. Claude creates an approval request in `Pending_Approval/`
2. Human moves the file to `Approved/`
3. Claude detects the approval and calls `send_email`
4. Result is logged and the approval file moves to `Done/`

The tool description includes this reminder so Claude is aware even without vault context.

## Troubleshooting

| Error | Fix |
|-------|-----|
| `token.json not found` | Run `python gmail_watcher.py` to complete OAuth flow |
| `credentials.json not found` | Download OAuth credentials from Google Cloud Console |
| `invalid_grant` | Delete `token.json` and re-run `gmail_watcher.py` |
| `insufficient permissions` | Ensure Gmail API scopes include `gmail.send` |

# Agent Skill: Human-in-the-Loop Approval

## Purpose
Enforce human approval for sensitive actions before the AI Employee executes them. This skill creates approval request files, monitors for decisions, and ensures no restricted action is taken without explicit human consent.

## When Approval Is Required

### Always Require Approval
| Action | Risk Level | Reason |
|--------|-----------|--------|
| Sending any email | High | External communication leaves the vault |
| LinkedIn posting | High | Public-facing content on behalf of company |
| Any payment action | High | Financial impact, irreversible |
| Deleting files | High | Data loss risk, violates "never delete" principle |
| Contacting new people | Medium | New external relationships need human judgment |

### Additional Triggers
- Any action flagged with emergency keywords (urgent, critical, asap) in Company_Handbook
- Bulk operations affecting more than 10 files
- System configuration changes
- Actions involving credentials or API keys
- Any action the AI Employee is uncertain about

### Auto-Approved (No HITL Required)
- Reading files
- Creating log entries
- Updating Dashboard
- Moving files between internal folders (except deletion)
- Standard file processing per FILE_PROCESSOR_SKILL

## Approval Request File

### Location & Naming
- Folder: Pending_Approval/
- Filename format: `APPROVAL_[action_slug]_[YYYY-MM-DD].md`
- Example: `APPROVAL_send_email_client_2026-02-16.md`

### File Format

```markdown
---
type: approval_request
action: [clear action description]
created: [YYYY-MM-DDTHH:MM:SSZ]
expires: [YYYY-MM-DDTHH:MM:SSZ - 24 hours after created]
status: pending
risk_level: low | medium | high
requested_by: claude_code
---

# Approval Request: [Action Title]

## Action Required
[Clear 1-2 sentence description of what the AI Employee wants to do and why.]

## Details
- **What will happen:** [Specific description of the action to be taken]
- **Target:** [Who or what is affected - recipient, file, system, etc.]
- **Risk level:** [low/medium/high with brief justification]
- **Reversible:** [Yes/No - can this action be undone?]

## Context
[Why this action is being requested. What triggered it. Any relevant background.]

## To Approve
Move this file to the **Approved/** folder.
The AI Employee will detect the approval, execute the action, and log the result.

## To Reject
Move this file to the **Rejected/** folder.
The AI Employee will log the rejection and skip the action.

## Expiry
This request expires at **[expiry timestamp]**.
If no decision is made by then, the request will be logged as timed out and the human will be prompted again if the action is still needed.

---
*Generated by AI Employee - HITL_APPROVAL_SKILL v1.0*
```

## How Claude Checks for Approvals

### Polling Process

```
1. Action requires approval
   ↓
2. Create APPROVAL_ file in Pending_Approval/
   ↓
3. Log the request (action_type: approval_requested)
   ↓
4. Update Dashboard with pending approval notice
   ↓
5. WAIT — Do not proceed with action
   ↓
6. On next invocation, check folders:
   ├─ Approved/ → File found? Execute action (Step 7a)
   ├─ Rejected/ → File found? Skip action (Step 7b)
   └─ Pending_Approval/ → Still there? Check expiry (Step 7c)
```

### Step 7a: Approved
1. Read the approval file from Approved/
2. Execute the approved action
3. Log the action with action_type: `action_executed` and note `approved_by: human`
4. Move approval file from Approved/ to Done/
5. Update Dashboard: add activity entry, decrement pending approvals

### Step 7b: Rejected
1. Read the rejection file from Rejected/
2. Do NOT execute the action
3. Log the rejection with action_type: `action_rejected` and note `rejected_by: human`
4. Move rejection file from Rejected/ to Done/
5. Update Dashboard: add activity entry, decrement pending approvals

### Step 7c: Expired (Still in Pending_Approval/)
1. Read the pending file
2. Parse `expires` timestamp from frontmatter
3. If current time > expires:
   - Update file status to `expired` in frontmatter
   - Log with action_type: `approval_expired`
   - Move file to Done/
   - Update Dashboard with expiry notice
   - If action is still needed, create a NEW approval request with fresh expiry
4. If not expired: Leave in place, continue waiting

## Dashboard Integration

When an approval request is created:
- Add to Recent Activity: `[HH:MM] Approval requested: [action description] (awaiting human decision)`
- Add/update line: `**Pending Approvals:** [count]`

When an approval is resolved:
- Add to Recent Activity: `[HH:MM] Approval [approved/rejected/expired]: [action description]`
- Decrement Pending Approvals count

## Safety Rules

1. **NEVER execute a sensitive action without a matching approval file in Approved/**
   - No exceptions. No shortcuts. No overrides.
2. **ALWAYS set an expiry time** — default 24 hours from creation
3. **ALWAYS log every decision** — requested, approved, rejected, expired
4. **NEVER modify an approval file** after creation — only move it between folders
5. **ONE approval per action** — each action gets its own approval file
6. **NEVER auto-approve** — the AI Employee cannot move its own files to Approved/
7. **CHECK Company_Handbook** — if in doubt about whether an action needs approval, it does
8. **PRESERVE audit trail** — all resolved approval files end up in Done/ for history

## Logging Integration

Use LOGGER_SKILL with these action types:
- `approval_requested` — when a new approval file is created
- `action_executed` — when an approved action is carried out
- `action_rejected` — when a human rejects an action
- `approval_expired` — when an approval times out

## Error Handling

- If Approved/ folder is inaccessible: log error, do not execute action, alert human
- If approval file is malformed: log error, create a new clean request
- If action fails after approval: log failure, keep approval in Done/ with failure note
- If multiple approval files exist for same action: use the most recent, log the conflict

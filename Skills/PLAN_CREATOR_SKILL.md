# Agent Skill: Plan Creator

## Purpose
Create structured Plan.md files for complex, multi-step tasks. Plans provide a roadmap with clear objectives, steps, timelines, and dependencies before execution begins.

## When to Create a Plan
- Any task requiring **3 or more distinct steps**
- Tasks spanning **multiple days or sessions**
- Tasks involving **external integrations** (email, APIs, services)
- Tasks requiring **human approval** at one or more stages
- Tasks with **dependencies** between steps
- Any Silver/Gold tier feature implementation

## Plan File Location
- Folder: Plans/
- Filename format: PLAN_[task_name]_[YYYY-MM-DD].md
- Example: Plans/PLAN_gmail_integration_2026-02-16.md

## Plan.md Format

```markdown
---
plan_name: [Descriptive Name]
created: [YYYY-MM-DD]
status: draft | active | completed | on_hold
priority: high | normal | low
estimated_completion: [YYYY-MM-DD]
owner: claude_code
requires_approval: true | false
---

# Plan: [Descriptive Name]

## Objective
[Clear 1-2 sentence description of what this plan achieves and why it matters.]

## Steps

- [ ] **Step 1: [Step Title]**
  - Description: [What needs to be done]
  - Dependencies: [What must be completed first, or "None"]
  - Estimated effort: [Time estimate]
  - Approval required: Yes/No

- [ ] **Step 2: [Step Title]**
  - Description: [What needs to be done]
  - Dependencies: [Step 1, external resource, etc.]
  - Estimated effort: [Time estimate]
  - Approval required: Yes/No

[Continue for all steps...]

## Timeline
| Phase | Steps | Target Date |
|-------|-------|-------------|
| [Phase name] | [Step numbers] | [YYYY-MM-DD] |

## Dependencies
- **Internal:** [Files, skills, or vault components needed]
- **External:** [APIs, services, credentials, human actions needed]

## Success Criteria
- [Measurable outcome 1]
- [Measurable outcome 2]

## Risks & Mitigations
| Risk | Impact | Mitigation |
|------|--------|------------|
| [Risk description] | High/Medium/Low | [How to handle it] |

## Notes
[Any additional context, references, or considerations.]

---
*Generated by AI Employee - PLAN_CREATOR_SKILL v1.0*
```

## Process Steps

### 1. Gather Requirements
- Understand the task scope from user request or inbox item
- Identify all sub-tasks needed
- Determine dependencies between steps
- Check Company_Handbook for approval requirements

### 2. Create Plan File
- Create file in Plans/ with correct naming convention
- Fill in YAML frontmatter with status: draft
- Write objective, steps, timeline, dependencies
- Mark steps requiring human approval

### 3. Update Dashboard
- Add entry to Recent Activity: `[HH:MM] Plan created: [plan_name]`
- Update System Status if this is a major initiative

### 4. Log Plan Creation
- Create log entry with action_type: plan_created
- Include plan filename and summary in log details

### 5. Notify for Approval
- If plan requires_approval: true, note in Dashboard
- Plan stays in draft status until human reviews
- Human changes status to active to begin execution

## Plan Status Lifecycle

```
draft → active → completed
  ↓        ↓
on_hold  on_hold
```

- **draft:** Plan created, awaiting review/approval
- **active:** Approved and in progress
- **on_hold:** Paused, waiting for external input or decision
- **completed:** All steps checked off and verified

## Updating Plans During Execution
- Check off steps as they are completed: `- [ ]` → `- [x]`
- Update status in frontmatter as plan progresses
- Log each step completion via LOGGER_SKILL
- Update Dashboard via DASHBOARD_UPDATER_SKILL

## Safety Rules

1. **NEVER skip approval steps** - If a step requires approval, wait for human
2. **ALWAYS create in draft status** - Plans need review before execution
3. **LOG plan creation** - Every plan gets a log entry
4. **UPDATE Dashboard** - Every plan creation updates Recent Activity
5. **PRESERVE existing plans** - Never overwrite; create new versions if needed

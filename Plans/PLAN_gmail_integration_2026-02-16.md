# Plan: Gmail Integration Setup

## Objective
Set up a Gmail watcher for Silver tier that monitors a designated inbox, extracts incoming emails, and converts them into task files in the Needs_Action folder for processing by the AI Employee.

## Steps

- [x] **Step 1: Configure Gmail API Credentials**
  - Description: Create a Google Cloud project, enable the Gmail API, generate OAuth 2.0 credentials, and store the credentials JSON securely in the vault. Set up the required scopes (gmail.readonly, gmail.modify) for inbox monitoring.
  - Dependencies: None
  - Estimated effort: 1 hour
  - Approval required: Yes

- [x] **Step 2: Build Gmail Watcher Script**
  - Description: Create `gmail_watcher.py` that authenticates with the Gmail API, polls the inbox at configurable intervals, and detects new unread emails. Include token refresh logic and error handling for network failures.
  - Dependencies: Step 1 (credentials must be in place)
  - Estimated effort: 3 hours
  - Approval required: No

- [x] **Step 3: Create Email-to-Task Converter**
  - Description: Build a module that takes raw email data (sender, subject, body, attachments, timestamp) and converts it into a standardized markdown task file with YAML frontmatter. Files are named `EMAIL_[subject_slug]_[YYYY-MM-DD].md` and placed in Needs_Action/. Map email priority headers and keywords to vault priority levels.
  - Dependencies: Step 2 (watcher must deliver email data)
  - Estimated effort: 2 hours
  - Approval required: No

- [x] **Step 4: Add Email Processing Skill**
  - Description: Create `Skills/EMAIL_PROCESSOR_SKILL.md` that defines how the AI Employee handles email-sourced tasks. Include rules for reply drafting (saved to Pending_Approval/), auto-categorization by sender/subject, and escalation of emails containing emergency keywords.
  - Dependencies: Step 3 (converter must produce task files)
  - Estimated effort: 1.5 hours
  - Approval required: No

## Risks & Mitigations
| Risk | Impact | Mitigation |
|------|--------|------------|
| OAuth token expiry during operation | Medium | Implement automatic token refresh with error retry |
| Gmail API rate limits hit | Low | Use exponential backoff; poll at reasonable intervals (every 60s) |
| Sensitive email data exposure | High | Store credentials outside vault repo; never log email body content |
| Malformed emails crash converter | Medium | Wrap parsing in try/catch; log errors and skip malformed emails |
| Google Cloud project approval delays | Medium | Start Step 1 early; have credentials ready before development |
